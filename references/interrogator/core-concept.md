# Core Concept Interrogation

<goal>
Understand the fundamental problem, solution, and reason this app should exist. No vague "it helps people" answers.
</goal>

---

## Question Bank

### The Problem

- What specific problem does this solve?
- Who has this problem? How do you know?
- How are people solving this problem today without your app?
- What's painful about current solutions?
- Is this a "vitamin" (nice to have) or "painkiller" (must have)?
- How often does this problem occur? Daily? Weekly? Once?
- What happens if the problem isn't solved?

### The Solution

- In one sentence, what does your app do?
- Why is software the right solution vs. a process/service/hardware?
- What's the core action a user takes in your app?
- If you removed everything except one feature, what would remain?
- How is this different from [competitor X]? Be specific.
- What would make someone switch from their current solution to yours?

### Validation

- Have you talked to potential users? How many?
- What did they say? Exact quotes if possible.
- Would they pay for this? How much?
- Have you seen anyone try to build this before? What happened?
- What's your unfair advantage in building this?

### Scope

- Is this a standalone app or part of something larger?
- What is explicitly OUT of scope for v1?
- What would v2 look like?
- What's the smallest possible version that delivers value?

- "That sounds like a feature. What's the underlying problem?"
- "How would you measure if this is working?"
- "'Better' than what? By what metric?"

---

## Exit Criteria

Category is complete when you can answer:
- [ ] What specific problem exists
- [ ] Who has it and how badly
- [ ] What the solution does in one sentence
- [ ] Why this solution vs alternatives
- [ ] What's in/out of scope for v1
