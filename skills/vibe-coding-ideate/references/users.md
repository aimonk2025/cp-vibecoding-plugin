# Users & Personas Interrogation

<goal>
Know exactly who uses this app, why, how, and what makes them different from each other. "Anyone can use it" is not a persona.
</goal>

---

## Question Bank

### User Types

- How many distinct types of users are there?
- What are they called? (Admin, Customer, Guest, etc.)
- Which user type is most important for v1?
- Are there users who use it but don't pay? Who pays?
- Are there internal users (your team) vs external users?

### For EACH User Type

- What's their job title / role / situation?
- What are they trying to accomplish?
- How tech-savvy are they? (Scale: grandma → developer)
- What device do they primarily use? Mobile? Desktop? Both?
- When do they use the app? (Morning routine? Emergency? Work hours?)
- How often do they use it?
- What's their alternative if your app doesn't exist?

### User Journey

- How do they discover the app?
- What's the first thing they do after signing up?
- What does "success" look like for them?
- What would make them stop using the app?
- What would make them recommend it to someone?

### Relationships

- Do users interact with each other? How?
- Is there a hierarchy? (Manager sees team's data, etc.)
- Can one user affect another user's experience?
- Are there teams/organizations or just individuals?

### Edge Users

- Who is explicitly NOT a target user?
- What if someone uses the app in an unintended way?
- Are there bad actors to consider? (Spammers, abusers)
- What about users with accessibility needs?

---

## Push-Back Phrases

- "'Everyone' is not a user. Who specifically?"
- "You said 'business users' — what kind? SMB? Enterprise? Freelancers?"
- "How do you know they want this? Have you asked?"
- "What if a power user and a casual user need different things?"

---

## Exit Criteria

Category is complete when you can answer:
- [ ] All user types identified and named
- [ ] Primary user for v1 defined
- [ ] Each user's goal, context, and frequency known
- [ ] User relationships/hierarchy mapped
- [ ] Edge cases and non-users identified
